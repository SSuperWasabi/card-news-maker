## 0. 배경 / 왜 하는가

- 단일 이미지 포스팅은 조회수 약 1만까지 나왔는데, **카드뉴스로 만들면 절반에도 못 미침.**
- **핵심 원인**: 캐러셀 1번 슬라이드 = 피드 썸네일인데, 기존 템플릿은 1번이 **검은 텍스트 카드**라 스크롤을 멈추게 하던 **강한 얼굴(에셋)이 2번으로 밀려남.**
- 따라서 **첫 카드를 "강한 비주얼 + 후킹 카피"로 전환**하는 것이 1순위. (캐러셀 리서치상 첫 슬라이드가 성과의 ~80% 좌우)

## 1. 확정 방향 ✅

1. **템플릿 택일 기능** — 메이커에서 생성 시 **① 기존 컨셉 / ② 카메라 UI(신규)** 중 택일.
2. **StudioBlank 디자인 시스템 유지** — ink `#0A0A0A` / paper `#FAFAFA` / stone(500 `#71717A`·300 `#A1A1AA`·100 `#E5E5E5`), **Inter**(300/700/800) + **IBM Plex Mono**, 직각(radius 0)·무그림자·1px 라인·모노크롬. 단색 차콜 레이어 OK, **그라데이션 금지.**
3. **규격**: 4:5 · **1080×1350 고정**. (※ 비율 동적화를 시도했으나, 표지 카메라 HUD — AF·FACE 박스, readout 등 — 가 4:5 세로 좌표 기준이라 다른 비율에서 레이아웃이 엉킴. 그래서 **4:5 고정 유지**. 향후 비율 가변이 필요하면 HUD 좌표를 카드 높이에 비례 조정하는 작업이 선행돼야 함.)
4. **신규 템플릿 = 카메라 UI, 5장 구성** (워크플로우·본문 압축 반영).

## 4. 구현 메모 (중요)

- **프레임워크 차이**: 원본 `.dc.html`은 datacosmos 포맷(`<x-dc>`, `image-slot.js`, `support.js`)으로 제작됨. **실제 메이커(`card-news-maker.src.html`)는 바닐라 단일 파일 JS** → 신규 템플릿은 메이커 방식(바닐라, html2canvas 캡처 호환)으로 **이식 구현**한다. (image-slot → 기존 이미지 처리 로직 재사용)
- **택일 UI** ✅: 에디터에 템플릿 선택(라디오: **기존 / 카메라 UI**) 추가. 기존 템플릿은 그대로 유지.
- **표지 이미지** ✅: **첫 플레이트 이미지를 표지에 재사용**(별도 슬롯 없음). 표지는 `cover`(풀블리드 크롭), 플레이트는 `contain`(무크롭)으로 같은 이미지를 다르게 렌더. → 1번 이미지 = 표지 + 플레이트01(잠정; 빌드 시 "표지전용 vs 플레이트에도 노출" 최종 확인).
- **Readout** ✅: `f/1.8 · 1/250 · ISO 100`, `● REC · AI-GEN`은 **고정 기본값(장식)** — 입력 필드 없음. (추후 편집 옵션 확장 여지)
- **신규 입력 필드**: 헤드라인(굵기 `**볼드**` 지원), 툴체인, 워크플로우 단계(툴+설명, 최대 3), 메인 본문, 해시태그. (CTA 칩 문구는 고정 `↓SAVE`/`+FOLLOW`)
- **재사용**: headline·chain·steps(tool/act)·insight·tags·이미지 처리·ZIP/PNG 캡처는 기존 로직 활용.
