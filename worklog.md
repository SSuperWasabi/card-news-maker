## 1. 배경 / 문제

- 단일 이미지 포스팅은 조회수 ~1만까지 나왔는데, 카드뉴스(캐러셀)로 만들면 절반에도 못 미침.
- **원인 진단**: 캐러셀의 1번 슬라이드 = 피드 썸네일인데, 기존 템플릿은 1번이 **검은 텍스트 카드**라 스크롤을 멈추게 하던 **강한 얼굴 에셋이 2번으로 밀려남**.
- 캐러셀 리서치: 첫 슬라이드가 성과의 ~80% 좌우. swipe-through(목표 65%+)·completion(55%+)·save/share가 핵심 지표.

## 2. 한 일 — "카메라 UI" 템플릿 신설 (택일)

기존 템플릿은 그대로 두고, 메이커에 **템플릿 택일(기존 / 카메라 UI)** 기능을 추가. 카메라 UI는 **6장 흐름**:

1. **표지 — 뷰파인더** (4:5 고정 1080×1350): 풀블리드 이미지 + 차콜 34% + 3분할 그리드 + 4모서리 브래킷 + 중앙 `AF·FACE`(브래킷+크로스헤어) + 모노 readout(`●REC AI-GEN`, `f/1.8 1/250 ISO100`) + 헤드라인(1줄 800 / 2줄 300) + 툴체인 + 푸터. 표지 이미지는 **첫 플레이트 이미지를 cover-crop** 재사용(html2canvas가 object-fit 무시 → canvas로 크롭 베이크).
2. **이미지 플레이트** — 기존 무크롭 Fit + 120px 캡션 바. 장수 조절.
3. **워크플로우(1장 통합)** — 모노 번호 + 툴명(Inter 700) + 설명(300), 1px 라인 구분, 최대 3단계.
4. **메인 본문** — `The Point` + Inter 300 본문 + `**키워드**` 볼드.
5. **The Prompt** — 사용한 프롬프트 전문(IBM Plex Mono). 에디터 신규 입력 필드.
6. **For More** — `One Prompt a Day.` 워드마크(마침표 40% 톤) + 룰드 테이블 소셜 4행(IG/FB/Threads/YouTube, 고정) + 푸터. 헤더·타이틀+디렉터리·푸터 3존 space-between.

### 주요 결정
- 템플릿 **택일로 추가**(기존 유지).
- 표지 이미지 = **첫 플레이트 재사용**(cover-crop). 첫 이미지가 표지(크롭)+플레이트01(전체)에 노출 — 의도된 reveal.
- readout(f/ISO/REC)는 **고정 장식값**.
- 엔딩 = `SAVE/FOLLOW` CTA 폐기(이미지 포스팅은 링크 안 눌림) → **The Prompt + For More 2장**으로 교체.
- 표지 **4:5 고정 유지**: 비율 동적화를 시도했으나 카메라 HUD(AF·FACE 등)가 4:5 좌표 기준이라 다른 비율에서 레이아웃이 깨짐 → 원복. (향후 가변 시 HUD 좌표 비례조정 선행 필요)

## 4. 빌드 / 사용

```bash
node _build-maker.js          # .src.html → .html (라이브러리 인라인)
```
그 다음 `card-news-maker.html`을 브라우저로 열기 → 좌상단 **템플릿** 선택 → 입력/이미지 드롭 → 카드별 PNG 또는 전체 ZIP 저장.
